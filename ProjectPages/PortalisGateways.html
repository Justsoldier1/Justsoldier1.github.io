Portalis gateways

Engine: Unity
Language: C#
Genre:Top down wave shooter
Contributors: Tyler Gledhill
Project origin: University Project
Project duration: 9 weeks
Weapons within my project are quite simple, as they mostly just change the parameters based around shooting. For example, the assault
rifle just increases the speed and damage of the projectiles, and increases the time between shots. Because these variables were
all natively set within the player controller, I could just make the weapon a switch case, which would set these parameters based
upon which weapon was selected. To make this simpler to use, I used integers to store which weapon was selected.
Showcase of weapon behaviours:

The most unique weapon within the arsenal included in the game is the rocket launcher. This is because it changes what prefab is
fired, which lets the bullet explode on contact, dealing area of effect damage.
Rocket launcher gameplay:
Since I had the vault system in mind, I decided to not include any form of inventory system within the game. Because of this, I needed
to create a separate way to change weapons. I decided to create weapon pickups, that would be created when the player throws out
their weapon. I added some velocity when this happens to further sell the idea of throwing out your weapon.
Within this project, weapons can be saved using the ‘Vault’, which is an area of the map accessible in between waves.
The vault area in my project:

The vault is made up of two game objects. These are vault borders, and shelf slots. Vault borders where added to prevent a bug, where
weapons could be thrown through walls that are not walkable, meaning that the player could easily lose their weapons within the
vault. Vault borders just check if a weapon enters its bounds using the OnTriggerEnter2D function, and sets it's position to the
centre of the vault.
Video showing the weapon throwing bug, and the vault borders in action:

The second game object used for managing the vault is a lot more important. These are the shelf slots, which handle holding the
objects, aswell as saving them to a file. When an object is thrown onto a shelf, it sets it's position to match the shelfs. It also has
it's spin effect stopped, as it would look strange to have the weapon be moving while stored onto a shelf.
Video showing what spinning weapons on a shelf would look like:

Then, the weapon is marked as being on a shelf, which means that it cannot despawn. The shelf is also set that it contains an item, so
that no other items can be placed onto the shelf. I then finally snap the object to the shelf's position, so that it is centered
properly.
The weapon have some velocity when thrown however, so they would move away from the shelf. Before, I created a coroutine that would
set the weapon back to the shelf position after 0.5 seconds, but this looked buggy and strange. Instead, I chose to manually reset
the velocity when it hits, which worked very well. I still kept the old system in case the object did not have a 2D rigid body
however.
Shelves holding weapons:

The weapons would then be stored while the game is in progress but will be deleted when the game is restarted. As the different
weapons just store different integers (see the weapon system section) I can just write this value to a text file. However, I could not
get the hang of reading multiple lines within a text file, so I needed to find a solution to store more than one weapon.
Image that displays the integer weapon saving system:
To create different weapons, I created an integer variable for the shelf. This would determine which file that the weapon data would
be contained inside. Then I could just construct the file path using string addition, and when using StreamWriter.WriteLine, it will
either write to a text file if there is one at that location, or create one there to be written into. I can then do the same thing
to read from this file when the map loads to place the necessary guns (if any at all) into the vault.
